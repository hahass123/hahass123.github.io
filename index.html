555构成的多谐振荡器会受电源电压影响

Posted on 2019-07-26 | In 模拟电路

在用555定时器搭建多谐振荡器的过程中遇到了一些课本上没说的问题，整理如下：
1、首先R1不能太小，太小会使OC门在放电时达到饱和，输出波形的低电平时间远大于计算结果。实测当R2为100Ω，C为几uF时，R1需要大于150Ω才能有正常波形。


2、实际电路当电源电压变化时，频率也会变化，而且变化相当大。所以需要稳压芯片稳压。原因嘛，就是供电电压大了，电容充放电的阈值就变大了，充电时间会变大，周期会变长，频率会变大。虽然树上公式推导能把VCC消掉，但实际测试确实影响很大，所以猜测电压阈值变大，并且电容充放电速率不会提高多少。

使用Hexo搭建属于自己的个性化博客

Posted on 2019-06-18

准备工作

下载node.js并安装（官网下载安装），此步骤会默认安装npm。
下载安装git（官网下载安装）

正式开始

在本地新建文件夹用来存放Hexo的相关文件和博客文章，下载安装hexo。方法：打开Git bash或者在文件夹右键打开，运行：
npm install -g hexo-cli（要翻墙）
hexo init Hexo
cd Hexo
npm install

都执行成功后，会在目录生成一个Hexo文件夹，以后Hexo文件夹就是本地静态网站配合和文章存放的地方。
接下来启动本地hexo，先本地预览hexo，执行如下命令：
hexo s

当看到命令行窗口出现，INFO Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.
的内容时，说明启动成功了，然后在浏览器输入网址：http://localhost:4000/ 即可看到本地hexo网站了。
无法访问http://localhost:4000/
命令行不要退出，不要按Ctrl+C
如果还不行：
如果您想要更改端口，或是在执行时遇到了 EADDRINUSE 错误，可以在执行时使用 -p 选项指定其他端口，如下：
$ hexo server -p 5000

将博客推送到GitHub和Coding

注册coding
新建项目
注意事项:
项目名要和用户名一致
要选择公开
创建好项目后在个人设置里面添加SSH公钥，可以参照帮助文档[配置 SSH 公钥访问代码仓库]http://dtid_0ecec7302fa20d1a.coding.me/ysusstech/

GitHub创建个人仓库
登录到GitHub,如果没有GitHub帐号，使用你的邮箱注册GitHub帐号：Build software better,
together 点击GitHub中的New repository创建新仓库，仓库名应该为：用户名.http://github.io
这个用户名使用你的GitHub帐号名称代替，这是固定写法，创建好项目后在个人设置里面添加SSH公钥，

打开本地的MyBlog文件夹项目内的_config.yml配置文件，将其中的type设置为git（注意冒号之后的空格，另外记得都用半角字符）
deploy:
type: git
repo:
coding: git@git.dev.tencent.com:dtid_0ecec7302fa20d1a/ysusstech.git
github: git@github.com:hahass123/hahass123.github.io.git
branch: master
要使用git链接，这样添加SSH公钥后每次提交不用输入账号密码

为 Hexo 安装 Git 插件
安装 hexo-deployer-git，否则会报 ERROR Deployer not found: git 的错误。
npm install hexo-deployer-git –save

hexo clean #清空hexo，主要删除Hexo根目录下的public文件夹
hexo g #重新成功public文件夹内容
hexo s #启动本地hexo服务
hexo d #发布到远程仓库

以后每次同步远程仓库，基本都是上面4个命令的顺序。

更改主题
既然默认主题很丑，那我们别的不做，首先来替换一个好看点的主题。
[官方主题]https://hexo.io/themes/：官方提供的各种主题
这里我推荐主题NexT。更换NexT主题可以参考[NexT]http://theme-next.iissnan.com/ 官方使用文档，点击开始使用一步一步操作。

写博客文章
在blog/source/_posts文件夹里写文章
命令行输入
$ hexo new [layout] <br>layout可选<br>| 布局（layout）|存储路径|说明<br>|-|-|-|-|<br>|post|source/_posts|默认，可以直接发布<br>|page|source|在source下新建一个文件夹<br>|draft|source/_drafts|新建文件将保持到_drafts中

可以用 publish 命令将草稿移动到 source/_posts 文件夹下
$ hexo publish [layout]

经过准备工作，现在差不多就可以安心的写作了。打开source/_posts文件夹下刚创建的Markdown文件，你会发现有如下内容：

title: hexo-blog
date: 2017-01-13 16:07:32

tags:

注意，这里的 title 是可以随便改的，当然了 date 也是可以改，不过没有改的必要；这里要说一下 tags 这个属性，如果需要配置多个有两种方式：

title: 文章标题
date: 2019-06-17 09:21:14
updated: 2019-06-17 09:21:14
comments: ture
tags:

    categories:
  • 分类
